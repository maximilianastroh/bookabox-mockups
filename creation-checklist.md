# Creation Checklist

> Pre-flight for new BookaBox work — before drafting, while drafting, and before sending. This is the shorter sibling of `review-checklist.md` — focused on getting the piece right the first time rather than auditing it after.

## Before drafting — answer these for yourself

### 1. What surface is this?
Website hero, OOH poster, Instagram post, email, app screen, kiosk, pitch deck slide, landlord letter? The surface determines almost everything else.

### 2. Which pole?
- **Pole A — Emotional / Edgy** — when you need to grab attention, build personality, or stand out against unrelated content
- **Pole B — Descriptive / Informative** — when intent is established and the job is to confirm, instruct, or persuade with evidence

If you cannot decide, ask the user **one** focused clarifying question and stop. Don't ask three.

### 3. Which language?
German (default, primary), English, or both? If both, do not just translate — recompose each in that language's natural voice.

### 4. Which audience?
- Consumer (Du)
- Landlord / B2B (Sie acceptable)
- Investor / institutional (Sie or English; formal claim variant if used)
- Press (factual, third-person constructions allowed)

### 5. Which value(s) is this piece foregrounding?
Pick at most two from: Einfachheit, Zuverlässig, Empathie, Gemeinschaft, Autonomie. The piece is about one or two of them, not all five.

### 6. What is the one thing the audience should remember?
If there are two things, pick one. The campaign color will emphasize exactly this thing.

## While drafting — keep these close

### Headline
- Short. Under 12 words is the target; under 8 is better for OOH.
- Pick one of the canonical patterns (see voice-and-tone.md):
  - HEADLINE
  - HEADLINE + SUBLINE
  - SKYLINE + HEADLINE
  - HEADLINE + "Logosatz"
  - Sentence A. Sentence B that reframes A. *(pitch decks)*
- For Pole A: aim for a setup → twist or an unexpected angle.
- For Pole B: state the offer or the fact directly. No twist.

### Body copy
- Sentences under 15 words where possible. Break long ones in two.
- No empty intensifiers (wirklich, absolut, 100%, beste, the best).
- No corporate buzzwords (Synergien, ganzheitlich, kundenzentriert, enabler).
- "Du" capitalized in German consumer copy. "Sie" only for landlord / B2B if the convention demands.
- Reuse phrasings and rhythms from the brand story (`brand-guide.md` § 1).

### Wordmark
- Spell it **"BookaBox.com"** — camelCase, with ".com". Every time.

### Claim
- Use **"Mach' Dir Dein Leben einfacher."** by default.
- Use **"Mach' Dir Dein Leben leichter."** for formal / institutional / investor contexts.
- Do not invent variants.

### Imagery brief (if visual)
- People → one of Motiv1–Motiv11 (see `imagery-index.md`). Never stock photos.
- Product → clean, real, branded surface visible.
- Color → black and grayscale base + one campaign color, applied to one element only.
- Typography → Silka. Black for headlines, Regular for body, Medium for labels.

## Before sending — silent self-review

Run the piece against this short list silently. Fix what fails before showing it to the user.

- [ ] Wordmark spelled correctly?
- [ ] Claim (if present) is one of the two sanctioned variants?
- [ ] "Du" / "Sie" consistent throughout?
- [ ] Pole consistent throughout (no Pole A and Pole B mixed)?
- [ ] Sentences short, no empty intensifiers, no buzzwords?
- [ ] Single focal point — one thing in the campaign color?
- [ ] Silka throughout (or fallback flagged as emergency)?
- [ ] If a headline pattern was used, does it actually match one of the canonical structures?

If two or more boxes fail, draft again before sending.

## Presenting the draft

When you show the user the draft, include a one-paragraph rationale:

- Which tonal pole you used and why
- Which headline pattern you picked
- Which value(s) the piece foregrounds
- Any deliberate brand choices the user should see (e.g., "I used the formal claim variant because this is investor-facing")

Keep the rationale short — three to four sentences max.

Offer **one alternative** only when it would genuinely help (e.g., "here's the same idea in the other tonal pole if you want to compare"). Do not flood with options.

## Gaps in the brand guide

If the brand guide is silent on something specific (an exact hex for a campaign background, an English voice parallel for a German idiom, an email signature template, an SMS tone), **say so explicitly**. Do not silently fill gaps with your taste.

When you hit a gap:
1. State that the guide does not specify this.
2. Propose a reasonable extension consistent with what the guide does specify.
3. Flag it so the user can decide whether to add the rule to the guide.
